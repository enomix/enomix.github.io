redis实现分布式缓存

1. 缓存概念

  1. 缓存是计算机内存中的一段数据

  2. 特点：读写快、断电丢失

  3. 作用：提高网站吞吐量和运行效率，主要是来减轻数据库访问压力

  4. 在哪里使用：一般在数据库中数据极少发生修改，更多用于查询操作时，可以使用缓存减轻数据库服务器压力

  5. 与分布式缓存的区别

    本地缓存：存储应用服务器内存中的数据为本地缓存（local cache）

    分布式缓存：存储在当前应用服务器之外的数据为分布式缓存（distribute cache）。比如说使用redis来存储缓存，这时候缓存是独立于应用服务器之外的，不占用应用服务器的内存。

    集群：将同一种服务的多个节点放在一起共同对系统提供服务的过程称之为集群（MySQL集群、Tomcat集群、redis集群）

    分布式：由多个不同服务集群共同对系统提供服务。共同组成的这个系统称之为分布式系统（distribute system）

  6. 利用MyBatis自身本地缓存结合redis实现分布式缓存

    1. MyBatis自身存在SqlSession（一级缓存），作用范围小，同一个会话关闭之后缓存立即丢失；

    2. MyBatis中也有应用级缓存（二级缓存）SqlSessionFactory，这个缓存是所有会话共享，当我们其他会话执行相同的查询可以利用这个二级缓存了（主要用这个）；单体应用使用二级缓存足够了，但是在集群中每一个服务对应一个jvm，本身缓存是不共享的，所以用redis来实现共享二级缓存。

    3. 如何开启：

      在映射文件XxxMapper.xml中添加一个<cache/>标签就可以开启缓存了，这个时候缓存存在于应用服务器内存中，这时我们称之为本地缓存

4. 缓存面试题

4.1. 缓存穿透

大量请求的key是不合理的，根本不存在缓存中和数据库中。这导致了大量非法请求直接到了数据库中，给数据库造成了巨大的压力，严重的可能会让数据库服务器宕机

fix：

  1. 做好参数校验，过滤非法key请求
  2. MyBatis当查询到非法数据时会返回null，并放入缓存中。redis就会有对应值为null的key，如果出现大量的请求这个值为null的key，redis就直接处理了，数据库压力就减轻了

4.2. 缓存击穿

请求的key存在于数据库中，缓存中不存在（可能因为缓存中的数据过期了），导致大量的请求直接到了数据库。

fix：

  1. 设置这些热点数据过期时间长一点 或 永不过期

4.3. 缓存雪崩

缓存在同一时间大面积的失效 或 缓存服务器宕机，导致大量的请求直接落到了数据库中，对数据库造成了极大的压力。

fix：

  1. 缓存宕机：

    redis集群

  2. 缓存同时失效：

    1. 给不同的业务设置不同的超时时间或是随机的超时时间
    2. 缓存永久存储（不推荐）
